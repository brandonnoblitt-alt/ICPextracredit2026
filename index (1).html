HOW I BUILT
SYNTAX BLASTER

THE IDEA

I needed to create a vocab review game for ICP class using 15 words from our word bank. I wanted it to feel like an actual arcade game, not just a boring quiz.

TOOLS USED

I used Claude (claude.ai) as my AI co-creator and VS Code to test the code locally before pushing to GitHub Pages.

THE PROCESS

I started with a starter prompt describing the game type, style, and requirements. Then I iteratively refined it — asking for retro arcade aesthetics, sound effects, score tracking, and animated transitions. I used the AI's own "suggest 3 UX feature ideas" prompt to get ideas like the progress bar, pixel-corner decorations, the rank system, a streak multiplier, power-ups, and an answer recap screen.

WHAT I LEARNED

Prompting an AI is a skill. Vague prompts get vague results. The more specific I was about visual style, mechanics, and layout, the better the output got. Iterating ("legiterating" 😅) was how the game went from plain to actually fun.

TECH STACK

Pure HTML, CSS, and JavaScript — no libraries, no frameworks. Hosted on GitHub Pages. Fonts loaded from Google Fonts. Sound generated with the Web Audio API so no external files needed.
